# Documentation Standards

- All documentation is written in English.
- Use clear headings, short paragraphs, and bullet lists. No emojis.
- Include only relevant sections — do not force empty headings.
- `docs/api.md` is the API reference. Update it if the API contract changes.
- `docs/overview.md` describes what is being built — not how.
- `docs/implementation-plan.md` is the canonical commit plan. Update it as decisions evolve.
- Code examples in docs must be syntactically correct and match the actual codebase patterns.
